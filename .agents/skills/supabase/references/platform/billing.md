## コスト管理

### Spend Cap（支出上限）

Pro プランではデフォルトで Spend Cap が有効。

- **有効**: 基本枠を超えるとサービスが制限される（追加課金なし）
- **無効**: 基本枠を超えると従量課金が発生（サービスは継続）

設定方法:
1. **Organization Settings → Billing → Spend Cap** に移動
2. 有効/無効を切り替え

### 使用量の確認

ダッシュボードの **Organization Settings → Billing → Usage** で現在の使用量を確認可能。

### アラート設定

使用量が閾値に達した際にメール通知を受け取れる。

## クレジット

- プラン変更時に未使用分のクレジットが発行される場合がある
- クレジットは次回以降の請求に自動適用される
- **Organization Settings → Billing → Credits** で確認可能

## 請求書

- 月次で請求書が発行される
- **Organization Settings → Billing → Invoices** からダウンロード可能
- PDF 形式で提供

## AWS Marketplace 連携

### メリット

- 既存の AWS アカウントで一元管理
- AWS の契約クレジットを利用可能
- 請求の統合

### セットアップ手順

1. AWS Marketplace で Supabase を検索
2. **Subscribe** をクリック
3. Supabase アカウントと連携
4. 以降の請求は AWS 経由で行われる

### 注意事項

- AWS Marketplace 経由の場合、Supabase ダッシュボードでの支払い方法設定は不要
- プラン変更は Supabase ダッシュボードから行う
- 請求は AWS の請求サイクルに従う

## ベストプラクティス

- 開発/ステージング環境には Free プランを活用する
- 本番環境では Spend Cap を無効にしてサービス中断を防ぐ
- 使用量アラートを設定してコストを監視する
- 不要なプロジェクトは一時停止または削除する
