# 成熟度モデル

## 概要

Supabase プロジェクトの成熟度を4段階で定義し、各レベルに応じた推奨設定を示す。

## Level 1: 個人開発 / プロトタイプ

### 特徴

- 個人またはごく少数のチーム
- プロトタイプや学習目的
- ユーザー数: 〜100

### 推奨設定

| 項目 | 設定 |
|------|------|
| プラン | Free |
| コンピュート | Micro |
| 環境 | 1環境（Local + Production） |
| バックアップ | 日次自動バックアップ（デフォルト） |
| RLS | 基本的なポリシーを設定 |
| 監視 | ダッシュボードで手動確認 |

### やるべきこと

- `supabase init` でプロジェクトを初期化
- 基本的な RLS ポリシーを設定
- マイグレーションファイルをバージョン管理
- ローカル開発環境をセットアップ

## Level 2: MVP / 小規模プロダクション

### 特徴

- 小規模チーム（2〜5人）
- 実ユーザーが利用するサービス
- ユーザー数: 100〜10,000

### 推奨設定

| 項目 | 設定 |
|------|------|
| プラン | Pro |
| コンピュート | Small 〜 Medium |
| 環境 | 2環境（Local + Production） |
| バックアップ | 日次 + PITR 検討 |
| RLS | 全テーブルに包括的なポリシー |
| 監視 | ダッシュボードレポート + アラート |

### やるべきこと

- Pro プランにアップグレード
- Spend Cap の設定を検討
- 接続プーリングを有効活用
- インデックスの最適化
- エラー監視の導入
- パフォーマンスベースラインの確立

## Level 3: 成長フェーズ

### 特徴

- 中規模チーム（5〜20人）
- 安定した成長を続けるサービス
- ユーザー数: 10,000〜100,000

### 推奨設定

| 項目 | 設定 |
|------|------|
| プラン | Pro または Team |
| コンピュート | Medium 〜 XL |
| 環境 | 3環境（Local + Staging + Production） |
| バックアップ | PITR 有効 |
| RLS | 包括的 + パフォーマンス最適化済み |
| 監視 | 外部監視サービス連携 |

### やるべきこと

- Staging 環境を追加
- CI/CD パイプラインでマイグレーションを自動化
- PITR を有効化
- リードレプリカの導入を検討
- 負荷テストの実施
- セキュリティ監査の実施
- カスタムドメインの設定
- ログドレインの設定

## Level 4: エンタープライズ

### 特徴

- 大規模チーム（20人以上）
- ミッションクリティカルなサービス
- ユーザー数: 100,000+
- コンプライアンス要件あり

### 推奨設定

| 項目 | 設定 |
|------|------|
| プラン | Team または Enterprise |
| コンピュート | XL 以上（オートスケーリング） |
| 環境 | 3+環境（Local + Dev + Staging + Production） |
| バックアップ | PITR + オフサイトバックアップ |
| RLS | 包括的 + 監査済み |
| 監視 | フル監視スタック |

### やるべきこと

- Enterprise プランの検討
- SSO / SAML の設定
- MFA の強制
- ネットワーク制限の設定
- PrivateLink の検討
- コンプライアンス要件への対応（HIPAA, SOC2 等）
- 災害復旧計画の策定と訓練
- SLA の確認と合意
- リードレプリカの導入
- Database Branching の活用
- セキュリティインシデント対応手順の策定
- 定期的なペネトレーションテスト

## レベルアップのチェックリスト

各レベルに進む前に、前のレベルの項目がすべて完了していることを確認する。段階的に成熟度を上げることで、安全かつ効率的にスケールできる。
