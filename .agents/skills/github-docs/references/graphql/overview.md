# GraphQL API 概要

## GraphQL とは

GitHub の GraphQL API は、必要なデータを正確に指定して取得できる柔軟なクエリ言語。REST API の複数リクエストを 1 回の呼び出しに統合でき、効率的なデータ取得が可能。

## エンドポイント

```
POST https://api.github.com/graphql
```

- すべての操作（クエリ・ミューテーション）で同一のエンドポイントを使用
- HTTP メソッドは常に POST（イントロスペクションクエリのみ GET 可）

## 主な特徴

| 特徴 | 説明 |
|------|------|
| **仕様ベース** | GraphQL 仕様に準拠した標準的な API |
| **強い型付け** | すべてのフィールドに型が定義されている |
| **イントロスペクション** | スキーマ自体をクエリして利用可能な型やフィールドを発見できる |
| **階層的** | クエリの構造が返される JSON データの構造と一致する |
| **アプリケーション層** | 特定のストレージエンジンに依存しない |

## REST API との違い

| 観点 | GraphQL | REST |
|------|---------|------|
| **エンドポイント数** | 単一エンドポイント | リソースごとに複数 |
| **データ取得** | 必要なフィールドのみ指定 | 固定されたレスポンス構造 |
| **リクエスト数** | 1 回のリクエストで複数リソースを取得可能 | リソースごとに個別リクエスト |
| **オーバーフェッチ** | 発生しない | 不要なデータも含まれる場合がある |
| **アンダーフェッチ** | 発生しない | 関連データに追加リクエストが必要な場合がある |
| **バージョニング** | スキーマの進化（バージョンなし） | URL ベースのバージョニング |

## スキーマ

GitHub GraphQL API のスキーマは以下の型で構成される:

- **Scalars**: プリミティブ型（String, Int, Boolean, ID など）
- **Objects**: フィールドを持つ複合型
- **Enums**: 列挙型
- **Interfaces**: 共有フィールド構造を定義する契約
- **Unions**: 複数のオブジェクト型を表現
- **Input Objects**: ミューテーションの入力パラメータ型

## イントロスペクション

スキーマ自体をクエリして、利用可能な型やフィールドを確認できる。

```graphql
query {
  __schema {
    types {
      name
      description
    }
  }
}
```
