# i18n

i18n（国際化）プラグインは、ユーザーのロケールに基づいて Better Auth が返すエラーメッセージの翻訳を可能にする。Better Auth はデフォルトで英語メッセージを提供するため、追加言語の翻訳のみが必要。

## セットアップ

### インストール

```bash
npm install @better-auth/i18n
```

### サーバー側

```typescript
import { betterAuth } from "better-auth"
import { i18n } from "@better-auth/i18n"

export const auth = betterAuth({
    plugins: [
        i18n({
            translations: {
                fr: {
                    USER_NOT_FOUND: "Utilisateur non trouvé",
                    INVALID_EMAIL_OR_PASSWORD: "Email ou mot de passe invalide",
                    INVALID_PASSWORD: "Mot de passe invalide",
                },
                de: {
                    USER_NOT_FOUND: "Benutzer nicht gefunden",
                    INVALID_EMAIL_OR_PASSWORD: "Ungültige E-Mail oder Passwort",
                    INVALID_PASSWORD: "Ungültiges Passwort",
                },
            },
        }),
    ],
})
```

## エラーレスポンス形式

```json
{
    "code": "INVALID_EMAIL_OR_PASSWORD",
    "message": "Email ou mot de passe invalide",
    "originalMessage": "Invalid email or password"
}
```

## ロケール検出戦略

### デフォルト（ヘッダーベース）

`Accept-Language` HTTP ヘッダーを品質値付きで解析:

```
Accept-Language: fr-CA, fr;q=0.9, en;q=0.8
```

### Cookie ベース

```typescript
i18n({
    translations: { /* ... */ },
    detection: ["cookie", "header"],
    localeCookie: "lang",  // デフォルト: "locale"
})
```

### セッションベース

```typescript
export const auth = betterAuth({
    user: {
        additionalFields: {
            locale: { type: "string", required: false },
        },
    },
    plugins: [
        i18n({
            translations: { /* ... */ },
            detection: ["session", "header"],
            userLocaleField: "locale",  // デフォルト: "locale"
        }),
    ],
})
```

### カスタム検出コールバック

```typescript
i18n({
    translations: { /* ... */ },
    detection: ["callback", "header"],
    getLocale: (ctx) => {
        if (!ctx.request) return null
        const url = new URL(ctx.request.url)
        return url.searchParams.get("lang")
    },
})
```

## 設定オプション

| オプション | 型 | デフォルト | 説明 |
|---|---|---|---|
| `translations` | `Record<string, Record<string, string>>` | 必須 | ロケールコード別の翻訳辞書 |
| `defaultLocale` | string | `"en"` または最初のロケール | 検出失敗時のフォールバックロケール |
| `detection` | `Array<"header" \| "cookie" \| "session" \| "callback">` | `["header"]` | 優先順位順の検出戦略 |
| `localeCookie` | string | `"locale"` | Cookie ベース検出の Cookie 名 |
| `userLocaleField` | string | `"locale"` | セッションベース検出のユーザーオブジェクトフィールド |
| `getLocale` | `(ctx) => string \| null \| Promise<string \| null>` | N/A | カスタムロケール検出関数 |

## サポートされるエラーコード

- `USER_NOT_FOUND`
- `INVALID_EMAIL_OR_PASSWORD`
- `INVALID_PASSWORD`
- `CREDENTIAL_ACCOUNT_NOT_FOUND`
- `EMAIL_NOT_VERIFIED`
- `SESSION_EXPIRED`

完全なリストはエラーコードリファレンスドキュメントを参照。

## 注意点

- 翻訳がない場合は組み込みの英語メッセージがデフォルト
- サポートされていないロケールは英語にフォールバック
- ロケールが検出されない場合は英語メッセージが使用される
- エラー以外のレスポンスは変更されない
- カスタムコールバックの `ctx.request` は非 HTTP 呼び出しで undefined になる可能性があるため、null セーフティチェックが必要
