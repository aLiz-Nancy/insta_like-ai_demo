# Migration

Guide for upgrading from Chakra UI v2 to v3, covering the automated codemod and manual breaking changes.

## Recommended Approach: Codemod

Run the automated codemod first — it handles the bulk of the migration:

```bash
npx @chakra-ui/codemod upgrade
```

The codemod automates:
- Component renames
- Prop modifications
- Import path updates
- Compound component restructuring

After running the codemod, apply the remaining manual changes described below.

## Removed Packages

Remove these dependencies from your project:

| Package | Replacement |
|---------|-------------|
| `@emotion/styled` | — (not needed in v3) |
| `framer-motion` | — (not needed in v3) |
| `@chakra-ui/icons` | `lucide-react` or `react-icons` |
| `@chakra-ui/hooks` | `react-use` or `usehooks-ts` |
| `@chakra-ui/next-js` | use `asChild` prop instead |

## Import Changes

**From `@chakra-ui/react`:**
`Alert`, `Avatar`, `Button`, `Card`, `Field`, `Table`, `Input`, `NativeSelect`, `Tabs`, `Textarea`, `Separator`, `useDisclosure`, `Box`, `Flex`, `Stack`, `HStack`, `VStack`, `Text`, `Heading`, `Icon`

**From local `components/ui` (generated by CLI):**
`Provider`, `Toaster`, `ColorModeProvider`, `Tooltip`, `PasswordInput`

## Key Component Changes

| Component | v2 | v3 |
|-----------|----|----|
| Toast | `useToast()` | `toaster.create()` with `type` instead of `status` |
| Modal | `<Modal>` | `<Dialog.Root>` |
| Button icons | `leftIcon` / `rightIcon` props | Direct children elements |
| Alert | Single component | `<Alert.Root>` + `<Alert.Content>` |
| Tooltip | `label` prop | `content` prop, import from `components/ui` |
