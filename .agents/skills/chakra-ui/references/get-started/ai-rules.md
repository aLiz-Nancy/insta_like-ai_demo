# AI Rules

Configuration rules for AI coding assistants (Cursor, GitHub Copilot, Claude) to generate accurate Chakra UI v3 code.

## Setup

Create a rules file in your project root:

- `.cursorrules` (Cursor)
- `.github/copilot-instructions.md` (GitHub Copilot)
- Or your preferred AI configuration file

Example frontmatter:

```yaml
---
description: Chakra UI v3 Development Rules
globs: "*.tsx"
alwaysApply: false
---
```

## Package Changes

**Remove these dependencies:**

- `@emotion/styled` and `framer-motion`
- `@chakra-ui/icons` — use `lucide-react` or `react-icons` instead
- `@chakra-ui/hooks` — use `react-use` or `usehooks-ts` instead
- `@chakra-ui/next-js` — use the `asChild` prop instead

## Import Sources

**From `@chakra-ui/react`:**

```
Alert, Avatar, Button, Card, Field, Table, Input, NativeSelect,
Tabs, Textarea, Separator, useDisclosure,
Box, Flex, Stack, HStack, VStack, Text, Heading, Icon
```

**From local `components/ui` (generated by CLI snippets):**

```
Provider, Toaster, ColorModeProvider, Tooltip, PasswordInput
```

## Critical Component Changes

| Component | v2 | v3 |
|-----------|----|----|
| Toast | `useToast()` | `toaster.create()` with `type` instead of `status` |
| Modal | `<Modal>` | `<Dialog.Root>` |
| Button icons | `leftIcon` / `rightIcon` props | Direct children elements |
| Alert | Single component | `<Alert.Root>` + `<Alert.Content>` |
| Tooltip | `label` prop | `content` prop; import from `components/ui` |

## Notes

- These rules are designed to prevent AI tools from generating v2 patterns when working on a v3 project
- Pair with [LLMs.txt](./ai-llms.md) files for comprehensive AI context
- The [MCP Server](./ai-mcp-server.md) provides an alternative, richer integration for AI assistants
